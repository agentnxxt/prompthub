Skills Guide SKILL.md

Building Agent Skills

AgentNXXT Hub · Updated March 2026 · 10 min read

Agent skills are modular capability packages that give AI coding agents new abilities on demand. Instead of baking everything into a system prompt, skills load dynamically when needed — keeping the agent focused and context-efficient.

1. What is a SKILL.md?

A SKILL.md file is the standard format for defining agent skills. It consists of two parts:

  1. YAML frontmatter — Metadata that tells the agent when to activate the skill
  2. Markdown body — Instructions the agent follows when the skill is invoked
---
name: deploy-to-coolify
description: Deploy applications to a Coolify instance
triggers:
  - deploy to coolify
  - spin up a service
  - launch on coolify
---

# Deploy to Coolify

You are a deployment agent that deploys applications
to a Coolify instance.

## Prerequisites
Before deploying, you need:
1. Coolify API URL
2. Coolify API Token
3. Server UUID

## Workflow
1. Validate inputs
2. Create the service via Coolify API
3. Start the service
4. Verify health

3. Writing Effective Skill Instructions

Skill instructions should be written as if you're briefing a skilled engineer on a task they haven't done before:

  • Be procedural — Numbered steps, not paragraphs of prose
  • Include templates — Provide exact API calls, config formats, code snippets
  • Handle errors — Tell the agent what to do when things go wrong
  • Set boundaries — What the skill should NOT do
  • Include verification — How to confirm the task succeeded

4. Skill Types

Code Generation Skills

Generate boilerplate, scaffold projects, create configs. These skills include templates and patterns.

## Template
When creating a new API endpoint:
1. Create handler at /api/v1/{resource}
2. Add route to router.go
3. Write request/response types
4. Add validation middleware
5. Write tests

Operational Skills

Deploy, monitor, debug. These skills make API calls and parse responses.

## Deployment Steps
1. Call POST /api/v1/services with compose
2. Poll GET /api/v1/services/{uuid} until healthy
3. If unhealthy after 5 min, check logs
4. Report URL and status to user

Analysis Skills

Review code, audit configs, assess risk. These skills define criteria and scoring.

## Review Criteria
- Security: Check for injection, auth bypass, secrets
- Performance: N+1 queries, missing indexes, blocking
- Style: Naming conventions, dead code, complexity

Research Skills

Search, compare, summarize. These skills define sources and output formats.
